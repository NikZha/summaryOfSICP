ПОСТРОЕНИЕ АБСТРАКЦИЙ С ПОМОЩЬЮ ПРОЦЕДУР

Тут сообщаются общие сведения о Лиспе, как языке созданного в лохматые года для задачи ИИ. Так же говорится о том, что лисп это интерпретируемый язык.

1.1. Элементы програмирования

Чтобы комбировать простое в сложное язык програмирования содержит:

  • Элементарные выражения, представляющие собой минимальные сущности с которыми язык имеет дело. Читай например типы даных (String, Integer и так далее).
  • Средства комбинирования, с помощью которых из простых элементов составляются сложные. Например структуры имеющие несколько полей разного типа.
  • Средства абстракции, с помощью которых сложные объекты можно обзывать и работать с ними как с единым целым. Например наследование и классы.

Далее он пишет, что от ЯП требуются средства комбинирования данных и процедур - то бишь создание объектов.

1.1.1. Выражения

На интерпретатор подаются выражения, а он в ответ выдаёт резутаты вычисления поданного выражения. Выражением может быть, как число так и какое-либо арифметическое выражение. По крайней мере эти выражения задействуются в этой подглаве.

Так же в этой подглаве даётся понятие записи выражения в лиспах и в схеме в частности, например


(- 10 2)

Что значит 10 - 2, так же они могут быть с вложенными скобами:


(* (- 10 8) (+ 1 4))

1.1.2. Имена и окружение

Оператор присваивания в scheme define позволяет задавать значение не только переменных, но и функций. Например:


(define pi 3.14)
(define radius 10)
(define circumference (* 2 pi radius))

1.1.3. Вычисление комбинаций

Вычисление подобного рода комбинаций происходит по принципу древовидной структруры, где ниже лежает листья самые левые:


(* (+ 2 (* 4 6))
(+ 3 5 7))

1.1.4. Составные процедуры

До этого мы рассматривали в лиспе (автор имеет в виду, то что диалект лиспа - схема):

  • Числа и арифметические операции представляющие собой элементарные данные и процедуры.
  • Вложение комбинаций дающее возможность комбинировать операции.
  • Объявление переменных.

Похожим образом объявляются процедуры (функции), например:


(define (square x) (* x x))

Чтобы запустить процедуру на выполнение надо написать в интерпретаторе:


(square 10)

Естественно, что процедуры можно комбинировать друг с другом:


(define (sum of squares x y)
(+ (square x) (square y)))

(define (f a)
(sum of squares (+ a 1) (* a 2)))

1.1.5. Подстановочная модель применения процедуры

В топике показывает в обучающих целях, как происходит подстановка.

1.1.6. Условные выражения и предикаты

Тут автор(ы) на примере языка scheme знакомят нас с условиями и предикатами:

cond

На примере модуля числа мелкая функция на примере cond:


(define (abs x)
    (cond ((< x 0) print (- x))
        (else (print x))))

if

Тоже самое на основе if: 


(define (abs1 x)
    (if (< x 0)
        (print (- x))
        (print x)))

and, or, not

Другие логические операнды позволяющие сделать выбор:


(define (more5 x)
    ((and (< 5 x) (> 10 x))))   
    
(define (>= x y)
    (or (> x y) (= x y)))

(define (>= x y)
    (not (< x y)))

1.1.7. Пример: вычисление квадратного корня методом Ньютона

Авторы исходя из их языка предлагают процедуру поиска путём рекурсии. Которая по пути дёргает ещё несколько функций.

Сам метод состоит в том, чтобы взять единицу, разделить на x - для получения приближение, потом выяснить частное, найти между ним и x среднее. Приравнять его к приближению вместо единицы и далее продолжить. Итеративное решение первая ф-ция. Далее я перенес решение из книги на js.

  • if нарушает это правило ради практичности

(define (sqrt-iteri guess x)
  (new-if (goodenough? guess x)

1.1.8. Процедуры как абстракции типа «черный ящик»

В топике даётся понимание процедуры как "чёрного ящика", то есть пользователя не должно волновать, как процедура функционирует внутри. Так же внутренние переменные процедуры не должны никак влиять на её использование. А так же даётся понятие блочной структуры на примере упаковки всех процедур в одну:


(define (sqrt x)
  (define (good-enough? guess)
    (< (abs (- (* guess guess) x)) 0.001))
  (define (improve guess)
    (average guess (/ x guess)))
  (define (average x y)
    (/ (+ x y) 2))
  (define (sqrt-iter guess)
    (if (good-enough? guess)
      guess
      (sqrt-iter (improve guess))))
  (sqrt-iter 1.0))

1.2. Процедуры и порождаемые ими процессы

Процедура представляет собой шаблон локальной эволюции (local evolution) вычис лительного процесса. Она указывает, как следующая стадия процесса строится из преды дущей.

Далее он собирается рассмотреть как процессы используют время и память.

1.2.1. Линейные рекурсия и итерация

Показывает как стём разворачивается и сворачивается на примере факторила 6:


(define (factorial n)
  (if (= n 0)
    1
    (* n (factorial (- n 1)))))

Однако автор приводит пример по форме рекурсивного, а по сути итеративной функции:


(define (factorial n)
  (fact-iter 1 1 n))

(define (fact-iter product counter max-count)
  (if (> counter max-count)
    product
    (fact-iter (* counter product)
      (+ counter 1)
      max-count)))

(print (factorial 100000))

Даже такой огромный пример scheme высчитывает благодаря оптимизации хвостовой рекурсии. Что делает этот пример итеративным. Однако такой же пример написанный на js приводит к падению, поскольку в js это рекурсия:


function fact(n) {
  return factIter(1, 1, n)
}

function factIter(product, counter, maxCount) {
    if(counter > maxCount) return product;
    return factIter(counter*product, ++counter, maxCount)
}

console.log(fact(100000))

Поддержка отимизации хвостовой рекурсии есть в стандарте ES6, но по факту существует только в сафари.

1.2.2. Древовидная рекурсия

Древовидную рекурсию дают на примере чисел фибоначи:


(define (fib n)
  (cond ((= n 0) 0)
    ((= n 1) 1)
    (else (+ (fib (- n 1))
        (fib (- n 2))))))

Само дерево вызовов понятно какое. 


                    fib 5
                   /     \
              fib 4       fib 3
             /    \       /    \
         fib 3   fib 2  fib 2  fib 1
        /    \   /   \  /   \
    fib 2  fib1 fib1 fib0 fib1 fib0
    /   \
 fib1 fib0

Интерпретатор scheme не оптимизирован для такого и потому вызов функции на небольших аргументах - проблема. Потому автор даёт сразу итеративный вариант в стиле рекурсивного варианта:


(define (fib1 n)
  (fib-iter 1 0 n))

(define (fib-iter a b count)
  (if (= count 0)
    b
    (fib-iter (+ a b) a (- count 1))))

Сами исходные файлы scheme и js находятся в каталоге.

Тут автор даёт информацию об O(n), как об оценке времени выполнения и памяти.

1.2.4. Возведение в степень

Цель топика показать, что одна и та же процедура можно занимать линейное время, как например процедура возведения в степень через рекурсивную формулу:


; bn = b bn−1
; b0 = 1 
(define (expt b n)
  (if (= n 0)
    1
    (* b (expt b (- n 1)))))

Оценка времени O(n), оценка памяти O(n).

Через итеративную процедуру с оценкой времени O(n) и оценкой памяти O(1):


(define (expt b n)
  (expt-iter b n 1))

(define (expt-iter b counter product)
  (if (= counter 0)
    product
    (expt-iter b
      (- counter 1)
      (* b product))))

Оценка памяти тут такая потому, что в scheme есть поддержка оптимизации рекурсии, где этого нет, там будет O(n).

Однако автор предлагает улучшить эту процедуру за счет:


b2 = b * b
b4 = b2 * b2
b8 = b4 * b4            
        

(define (fast-expt b n)
  (cond ((= n 0) 1)
    ((even? n) (square (fast-expt b (/ n 2))))
    (else (* b (fast-expt b (- n 1))))))

(define (square x)
  (* x x))

(define (even? n)
  (= (remainder n 2) 0))

Таким образом сложность по времени будет Olog(n), что существенно быстрее чем O(n) на больших числах.
